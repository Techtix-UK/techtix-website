Platform Overview

Welcome to the official documentation for OmniDeploy IT. OmniDeploy is a hyper-efficient, fully integrated IT management platform engineered by Techtix. It is designed specifically to bring Helpdesk Ticketing, IT Asset Management (ITAM), Contact Directory, and Mobile Device Management (MDM) into a single, unified interface.

OmniDeploy uses a strict role-based architecture consisting of three primary interaction tiers:

  • End-Users / Clients: Access the public-facing portals to submit tickets, review active requests, read FAQ articles, and sign hardware agreements.
  • Helpdesk Agents: Access the core /staff portal to resolve tickets, manage assets, check-in hardware via barcode scanning, and utilize canned responses.
  • SuperAdmins: Manage global settings, email API configurations, agent accounts, software deployment packages, and platform licensing.

End-User Guide

The End-User experience is designed to be entirely frictionless. Users do not need a preexisting account to submit a request, though creating one provides enhanced visibility into their history.

The Client Portal

Users navigate to /portal (or the root domain) to access the support center. If they are a first-time user, they can submit a "Guest" ticket, or click Create Account / Set Password.

Seamless Account Linking: If an IT Agent has already added a user profile to the Directory (e.g., during onboarding), the user simply needs to register using that exact email address. OmniDeploy will automatically map their new login to their existing profile, instantly granting them access to their historic tickets and assigned assets.

Submitting Tickets

Users can submit issues to the IT department in two distinct ways:

  1. Web Portal: By clicking "+ New Ticket", the user selects a specific Help Topic, enters a subject, provides details, and uploads relevant screenshots or error logs.
  2. Email Integration: If SuperAdmins have configured an inbound mailbox (e.g., helpdesk@company.com), users can simply email their issue. The OmniDeploy backend will parse the email, strip out signatures, download attachments, and automatically generate a ticket under the "General" default category.

Hardware Handover (EULA)

OmniDeploy eliminates paper trails for hardware assignments. When IT provisions a new asset (like a laptop or phone) to a user, they require a signature.

The user navigates to the dedicated endpoint: /eula?asset=[ASSET_TAG]. This can be done by clicking a link emailed to them, or by scanning the QR code physically affixed to the device.

On this page, the user reviews the company's Acceptable Use Policy and physically signs the screen using their mouse or touchscreen. Upon submission, OmniDeploy generates a timestamped, unalterable digital receipt and binds it permanently to the asset's audit history.

IT Agent Guide

The Staff Portal (/staff) is the engine room of OmniDeploy IT. Helpdesk Agents log in here to manage the day-to-day operations of the IT environment.

Ticket Queues & Management

The Tickets tab is the default landing page. It provides real-time metrics (Open, Unassigned, High/Emergency, Closed Today) and allows agents to filter via four main queues: All Open, My Tickets, Unassigned, and Closed.

Clicking a ticket opens the comprehensive management modal:

  • Ticket Thread: Displays the entire conversation history, including system audit trails (e.g., "Status changed: OPEN -> IN PROGRESS").
  • Internal Notes: Agents can post hidden notes to a ticket. These are styled differently in the thread and are strictly invisible to the End-User.
  • CC Emails: Agents can add comma-separated email addresses to loop external vendors or managers into the automated notification cycle.
  • Attachments: Files uploaded by the user or via email are securely stored and can be downloaded with a single click.

Ticket Collision & Locking

To prevent two agents from answering the same ticket simultaneously, OmniDeploy features an automated Ticket Lock system.

When Agent A opens a ticket, the system locks it to their account. If Agent B opens that exact same ticket, Agent B will see a bright yellow warning banner stating: "⚠️ Warning: Agent A is currently viewing/editing this ticket!"

Asset Inventory

The Asset grid maintains a live record of all hardware. It supports deep searching by Asset Tag, Serial Number, User, Make, or Model.

Inside an Asset profile, agents can track deep technical specifications (OS, CPU, RAM, MAC/IP Address, Storage Drives) as well as remote support credentials (AnyDesk ID and Password). You can also upload related documentation (like purchase invoices or warranty certificates) directly to the asset.

Dispatch & Quick Actions

The Dispatch tab is designed for rapid response during phone calls or physical walk-ins.

  • Raise Ticket (Dispatch): Type a user's name or phone number. OmniDeploy will query the Directory, pull their profile, and automatically populate their assigned devices in a dropdown so you can log a ticket on their behalf in seconds.
  • Hardware Check-In/Out: Click the "📷 Scan" button to activate the device's webcam or smartphone camera. Scan an asset's barcode to instantly reassign it, check it into stock, or check it out to a user.

Directory Management

The Directory houses Profiles for both Users and Companies (useful for Managed Service Providers supporting multiple businesses).

Opening a profile provides a 360-degree view of that entity, displaying all contact information, a list of all currently assigned hardware, and a history of their 10 most recent support tickets.

SuperAdmin Guide

SuperAdmins have exclusive access to the Settings (Admin) tab, which dictates the global behavior, routing, and security of the entire OmniDeploy instance.

Email Integration (IMAP/Graph)

OmniDeploy can act as a fully functional email client, transforming inbound emails into tickets and sending outbound HTML-formatted replies.

  • SMTP (Outbound): Required for the system to send notifications, ticket replies, and EULA receipts. Enter your mail server host, port, user, and password.
  • Inbound - Standard IMAP: Legacy protocol supported by most basic web hosts and Gmail. Requires the IMAP Host, Port (usually 993), User, and an App Password.
  • Inbound - Microsoft Graph API: The modern, highly secure protocol required for Office 365 and Azure environments (since Microsoft disabled Basic Auth). You must provide the Azure Tenant ID, Client ID, and Client Secret.

    Configuration Note: The Azure App Registration must be granted the Mail.ReadWrite Application Permission, and an Azure Administrator must click "Grant admin consent for [Organization]".

Classifications & Help Topics

Customize the platform's data taxonomies to fit your organization.

  • Asset Categories: A comma-separated list populating the dropdown when creating hardware (e.g., Laptop, Server, Switch, Printer).
  • Help Topics: Define routing categories for tickets. Topics marked as Private will be completely hidden from the End-User portal, allowing IT staff to categorize backend/infrastructure tickets without cluttering the public UI.

Dynamic Canned Responses

SuperAdmins can pre-write standard replies (e.g., "Password Reset Instructions", "Reboot Required") to ensure standardized communication.

Canned responses support dynamic variable injection. When an agent selects a response, OmniDeploy will automatically replace tags before inserting the text into the reply box. Available tags are:

{ticket_id} // Injects the current ticket number (e.g., TKT-10492)
{user_name} // Injects the requester's full name
{agent_name} // Injects the replying IT Agent's name

MDM & Software Deployment

If your organization utilizes the OmniDeploy Desktop Agent (installed locally on Windows endpoints), SuperAdmins can register software packages via the backend database.

Once a package is registered, any IT Agent can open an Asset profile, select the software from the "Remote Software Deployment" dropdown, and click Deploy. OmniDeploy will queue the task, and the desktop agent will silently download and execute the installer during its next polling cycle.

Staff Management

SuperAdmins control access to the /staff portal. You can create accounts, assign Full Names, and designate roles.

Roles:

  • Helpdesk: Standard access. Cannot see or access the Settings tab.
  • SuperAdmin: Full access, including global settings, API keys, and Staff account creation.

Knowledge Base (FAQ)

The Knowledge Base (KB) module allows admins to publish rich-text articles to the End-User portal to encourage self-service and deflect common tickets. Articles can be saved as Drafts (hidden) while being written, and then toggled to Published to make them instantly live.

Licensing & Activation

OmniDeploy IT requires a valid cryptographic license key to operate. The license dictates your expiration date and which modules (Tickets, Assets, Directory, Settings) are visible in the navigation bar.

License Expiration: If your license expires, or if an invalid key is applied, the OmniDeploy staff portal will immediately lock down. All agents will be redirected to a blank activation screen until a valid key is provided.

Applying a License Key

  1. Log in as a user with the SuperAdmin role.
  2. Navigate to the Settings (Admin) tab.
  3. Click on the Licensing pill-tab.
  4. Paste your new HMAC-SHA256 signature into the "Update License Key" input box.
  5. Click Apply Key. The portal will verify the cryptographic signature against your master secret and automatically reboot the interface.

If you need to extend your operational timeframe or unlock additional platform modules, please contact your Techtix Account Manager to generate a new key.
